\section{Definitions}

We define non-empty paths, possibly empty paths, module names and declaration names:

\begin{align*}
    P     & ::= X \;|\; X.P                   \\
    \pi   & ::= \epsilon \;|\; P              \\
    M     & ::= P \;|\; P[\overline{\tx = t}] \\
    \Next & ::= N \;|\; \tx
\end{align*}

We redefine type environments:
$$\Gamma ::= \overline{D}; \overline{D}$$

The environment contains a module environment followed by local declarations (both can be empty).
We will denote: $\Gamma = \Gammatop; \Gammabot$ when we need to distinguish both parts.

We may also denote the empty environment $\epsilon$ instead of $\epsilon; \epsilon$. \\

We define $\name: \left\{\begin{array}{ll}
        X : T                    & \mapsto X   \\
        x : \bigcap \overline{T} & \mapsto x   \\
        \tx : \star              & \mapsto \tx \\
        \tx : [=t]               & \mapsto \tx
    \end{array}\right.$  \\

We consider behavior environments and declaration lists as partial functions
from $\BX$ and $\Next$ to $\Beh$ and $D$, respectively.
If the name appears multiple time in the list, the right-most occurrence is returned, as one would expect.

We also define:

$$
    (\Gammatop; \Gammabot)(\Next) = \left\{\begin{array}{l}
        \Gammabot(\Next) \text{ if } \Next \in \dom(\Gammabot) \\
        \Gammatop(\Next) \text{ otherwise }
    \end{array}\right.
$$

and

$$\Gamma(X.P) = \Gamma(X)(P)$$

\section{Type environments}

\subsection{Local environments}

We define two notations: $\Gamma[P \leftarrow T]$ and $\Gamma[\pi \texttt{+=} D]$ as follows.

\begin{figure}[h]
    \[
        \begin{array}{ll}
            \Gamma[X \leftarrow T]                                               & = \Gamma, X : T                              \\
            \left(\Gamma, X : T' \right)[X.P \leftarrow T]                       & = \Gamma, X : \{ T'[P \leftarrow \{ T \}] \} \\
            \left(\Gamma, D\right)[X.P \leftarrow T] \text{ if } X \neq \name(D) & = \Gamma[X.P \leftarrow T], D                \\
        \end{array}
    \]
    \caption{Notation $\Gamma[P \leftarrow T]$}
\end{figure}


By abuse of notation the above definition has used:
\[
    \begin{array}{ll}
        \left(\Moddecl\right)[P \leftarrow T]                                      & =  \Moddecl[\overline{D}[P \leftarrow T]]                                     \\
        \left((\overline{\tx : \star}) \rightarrow \Moddecl\right)[P \leftarrow T] & = (\overline{\tx : \star}) \rightarrow \Moddecl[\overline{D}[P \leftarrow T]] \\
    \end{array}
\]

\begin{figure}[h]
    \[
        \begin{array}{ll}
            \Gamma[\epsilon \texttt{+=} \{ X : T \}]                                & = \Gamma[X \leftarrow T]                                          \\
            \left(\Gamma, X : \{ \overline{D'} \}\right)[X \texttt{+=} D]           & = \Gamma, X : \{ \overline{D'}, D \}                              \\
            \left(\Gamma, X : \{ \overline{D'} \}\right)[X.P \texttt{+=} D]         & = (\Gamma, X : \{ \left(\overline{D'}\right)[P \texttt{+=} D] \}) \\
            \left(\Gamma, D'\right)[X.P \texttt{+=} D] \text{ if } X \neq \name(D') & = (\Gamma[X.P \texttt{+=} D], D')
        \end{array}
    \]
    \caption{Notation $\Gamma[\epsilon \texttt{+=} D]$}
\end{figure}

Note that $\Gamma[X.P \leftarrow T]$ and $\Gamma[X.\pi \texttt{+=} D]$ are not defined if $X \notin \dom(\Gamma)$ and
$\Gamma[\epsilon \texttt{+=} D]$ is not defined if $D$ is not $X : T$.
That's because only modules can be exported at the toplevel, and it's not possible to add a declaration to a module that does not exist.

We will denote $P; \Sigma; \Gamma \vdash \overline{T}$ to say that all types in $\overline{T}$ are
well-formed. Note that $P; \Sigma; \Gamma \vdash \overline{D}$ will NOT mean the same for declarations (see next section).

The above notations and definitions allow us to propose the following rules for well-formedness of type environments:

\[
    \begin{prooftree}
        \infer0{ \pi; \Sigma; \Gammatop \vdash \epsilon }
    \end{prooftree}
\]

\[
    \begin{prooftree}
        \hypo{ P; \Sigma; \Gammatop \vdash \Gammabot }
        \infer1{ P; \Sigma; \Gammatop \vdash \Gammabot, \tx : \star }
    \end{prooftree}
\]
\[
    \begin{prooftree}
        \hypo{ P; \Sigma; \Gammatop \vdash \Gammabot }
        \hypo{ \Sigma; \Gammatop; \Gammabot \vdash t }
        \infer2{ P; \Sigma; \Gammatop \vdash \Gammabot, \tx : [= t] }
    \end{prooftree}
\]
\[
    \begin{prooftree}
        \hypo{ P; \Sigma; \Gammatop \vdash \Gammabot }
        \hypo{ P; \Sigma; \Gammatop; \Gammabot \vdash \overline{T} }
        \infer2{ P; \Sigma; \Gammatop \vdash \Gammabot, x : \bigcap \overline{T} }
    \end{prooftree}
\]
\[
    \begin{prooftree}
        \hypo{ \pi; \Sigma; \Gammatop \vdash \Gammabot }
        \hypo{ \pi.X; \Sigma; \Gammatop[\pi.X \leftarrow \{\}]; \Gammabot \vdash T }
        \infer2{ \pi; \Sigma; \Gammatop \vdash \Gammabot, X : T }
    \end{prooftree}
\]

The attentive reader will have noticed that $T$ does have access to the local environment $\Gammabot$
in the last rule. This is intended. We will drop the local environment only when checking well-formedness
of $\Moddecl$.

When we check $\overline{\tx : \star}$, we use an empty type environment, because we want to allow the

does not need the type environment anyway.
